# Changes — 2026-02-27

Audit against Claude Code 2.1.50+. Identified 15 missing features + code quality issues. Applied fixes described below.

---

## Fixes Applied

### Security

**`src/tools/bash.mjs` — Heredoc delimiter smuggling prevention**
Added `validateHeredocs(command)` that scans all heredoc blocks and rejects any
where the body contains the delimiter on its own line (would terminate the heredoc
prematurely, enabling command injection). This mirrors CC 2.1.38's fix.

### Bug Fixes / Code Quality

**`src/utils/summarize.mjs` — Remove hardcoded model**
Replaced hardcoded `'claude-haiku-4-5-20251001'` with a `getSummarizationModel()`
resolver that checks (in order):
1. `OPENCLAUDE_COMPACT_MODEL` env var
2. `compactModel` in user config
3. Falls back to the dated haiku ID

This prevents breakage when the model is deprecated.

### New Features

**`src/tools/skills-discovery.mjs` — Skill hot-reload (CC 2.1.0 parity)**
Added:
- `startSkillsHotReload(projectDir)` — watches `.claude/skills/` dirs with `fs.watch`
- `stopSkillsHotReload()` — tears down watchers
- `onSkillsChanged(callback)` — register a listener for file-change events

The TUI can register a cache-invalidation listener so skill changes are visible
immediately without a session restart.

**`src/agents/named-agents.mjs` — `isolation` frontmatter key**
Agent definitions now parse and expose `isolation: worktree` (defaults to `'none'`).
The execution layer can check `agent.isolation === 'worktree'` to spin up an isolated
git worktree via the new utility below.

**`src/agents/worktree-isolation.mjs` — New file**
New module implementing git worktree isolation for agents (CC 2.1.50 parity):
- `createAgentWorktree(agentName, projectDir)` — creates a temp branch + worktree
- `cleanup({ keepIfChanged })` — auto-removes if no changes; preserves if agent made changes
- `runInWorktree(agentName, taskFn, projectDir)` — high-level helper

**`src/plan/plan.mjs` — Context-clear on plan acceptance (CC 2.1.x parity)**
Added `onPlanApproved(callback)` registration system. When `approvePlan()` is called,
all registered callbacks are fired asynchronously. The TUI should register a
`compactMessagesWithAi` call here so the planning conversation is summarised before
execution begins, freeing the full context window.

**`src/cli/commander-setup.mjs` — `claude agents` subcommand (CC 2.1.50 parity)**
Added `claude agents [--json]` subcommand that lists all configured agents from
`~/.claude/agents/` and `.claude/agents/` in a formatted table. Mirrors
`claude agents` in CC 2.1.50.
