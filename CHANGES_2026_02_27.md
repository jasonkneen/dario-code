# Changes — 2026-02-27

Audit against Claude Code 2.1.50+. Identified 15 missing features + code quality issues. Applied fixes described below.

---

## Fixes Applied

### Security

**`src/tools/bash.mjs` — Heredoc delimiter smuggling prevention**
Added `validateHeredocs(command)` that scans all heredoc blocks and rejects any
where the body contains the delimiter on its own line (would terminate the heredoc
prematurely, enabling command injection). This mirrors CC 2.1.38's fix.

### Bug Fixes / Code Quality

**`src/utils/summarize.mjs` — Remove hardcoded model**
Replaced hardcoded `'claude-haiku-4-5-20251001'` with a `getSummarizationModel()`
resolver that checks (in order):
1. `OPENCLAUDE_COMPACT_MODEL` env var
2. `compactModel` in user config
3. Falls back to the dated haiku ID

This prevents breakage when the model is deprecated.

### New Features

**`src/tools/skills-discovery.mjs` — Skill hot-reload (CC 2.1.0 parity)**
Added:
- `startSkillsHotReload(projectDir)` — watches `.claude/skills/` dirs with `fs.watch`
- `stopSkillsHotReload()` — tears down watchers
- `onSkillsChanged(callback)` — register a listener for file-change events

The TUI can register a cache-invalidation listener so skill changes are visible
immediately without a session restart.

**`src/agents/named-agents.mjs` — `isolation` frontmatter key**
Agent definitions now parse and expose `isolation: worktree` (defaults to `'none'`).
The execution layer can check `agent.isolation === 'worktree'` to spin up an isolated
git worktree via the new utility below.

**`src/agents/worktree-isolation.mjs` — New file**
New module implementing git worktree isolation for agents (CC 2.1.50 parity):
- `createAgentWorktree(agentName, projectDir)` — creates a temp branch + worktree
- `cleanup({ keepIfChanged })` — auto-removes if no changes; preserves if agent made changes
- `runInWorktree(agentName, taskFn, projectDir)` — high-level helper

**`src/plan/plan.mjs` — Context-clear on plan acceptance (CC 2.1.x parity)**
Added `onPlanApproved(callback)` registration system. When `approvePlan()` is called,
all registered callbacks are fired asynchronously. The TUI should register a
`compactMessagesWithAi` call here so the planning conversation is summarised before
execution begins, freeing the full context window.

**`src/cli/commander-setup.mjs` — `claude agents` subcommand (CC 2.1.50 parity)**
Added `claude agents [--json]` subcommand that lists all configured agents from
`~/.claude/agents/` and `.claude/agents/` in a formatted table. Mirrors
`claude agents` in CC 2.1.50.

**`src/memory/auto-memory.mjs` — Auto Memories (CC 2.1.32 parity)**
New module for cross-session fact persistence in `.claude/memory/`:
- `saveMemory(key, value, source, cwd)` — writes YAML-frontmatter markdown files
- `loadMemories(source, cwd)` — loads all stored memories
- `deleteMemory(key, source, cwd)` / `clearMemories(source, cwd)` — removal helpers
- `buildMemoryContext(cwd)` — formats memories section for injection into system prompt
- `shouldExtract(messages, lastExtractTime)` — rate-limiting heuristic (>5 new turns OR >10 min)
- `extractMemories(messages, cwd)` — AI-powered extraction returning `[{key, value}]`

**`src/memory/memory-watcher.mjs` — Background memory watcher**
- `startMemoryWatcher(getMessages, cwd)` — periodic extraction every 5 minutes via `setInterval(...).unref()`
- `stopMemoryWatcher()` / `triggerExtraction()` exports

**`src/core/config.mjs` — Memory context injection**
`getSystemPrompt()` now dynamically imports `buildMemoryContext` and appends a `# Memories`
section to the system prompt when stored memories exist. Silent on failure.

**`src/utils/summarize.mjs` — "Summarize from here" (CC 2.1.32 parity)**
Added `compactFromMessage(messages, fromIndex, keepLastN = 4)` that summarizes
`messages[fromIndex ... -keepLastN]` and splices a summary message into that slot,
keeping messages before `fromIndex` and the most recent `keepLastN` verbatim.

**`src/cli/commands.mjs` — `/compact <N>` and `/memory` commands**
- `/compact <N>` — compact from message N onwards
- `/compact last <N>` — keep only last N messages, summarize prior messages
- `/memory list` — list auto-extracted memories
- `/memory edit <key>` — open memory file in `$EDITOR`
- `/memory delete <key>` / `/memory clear` — remove stored memories

**`src/keyboard/history-search.mjs` — Bash history Tab autocomplete (CC 2.1.14 parity)**
Added `getTabCompletion(partial)` to `HistorySearch` class: walks history in reverse
to find the most recent entry starting with `partial`.

**`src/keyboard/index.mjs` — Tab handler wired to history**
`KeyboardManager` gains `setCurrentInput(input)` and `_getTabCompletion(partial)`.
Tab key now emits `'tab-complete'` with the matched command when a history match exists,
falling back to `'toggle-thinking'` when no match is found.

**`src/sandbox/sandbox.mjs` — Linux sandbox support (CC 2.1.41 parity)**
- `isSandboxSupported()` now returns true on Linux when `bwrap` or `firejail` is found
- `detectLinuxSandboxBin()` — `spawnSync('which', [...])` probe for bubblewrap/firejail
- `wrapCommandLinux(command, projectDir)` — wraps commands with bwrap or firejail
- `wrapCommand` / `executeWithSandbox` updated to branch on `platform()`

**`src/tools/executor.mjs` — Diff preview in permission prompts (CC 2.x parity)**
`createInteractivePermissionHandler()` now calls `_showDiffPreviewIfApplicable(tool, input, output)`
before the allow/deny prompt. For `Write` tools it diffs old vs new file content;
for `Edit` tools it diffs `old_string` vs `new_string`. Uses `generateUnifiedDiff` +
`renderDiffColored` from `utils/diff.mjs`. Best-effort — never blocks the prompt.

**`src/plugins/installer.mjs` — Plugin SHA pinning (CC 2.1.14 parity)**
- `installFromGit(source, { pin })` — clones repo and checks out the specified SHA/ref
- Validates pin ref (alphanumeric + `._-/` only) to prevent shell injection
- Records `resolvedSha` (actual HEAD after checkout) in installed manifest
- `updatePlugin(name, { force, unpin })` — skips update for pinned plugins unless forced

**`src/plugins/manifest.mjs` — SHA pin schema**
Schema documents `pin` and `resolvedSha` fields. `validateManifest` validates
`pin` as a non-empty string when present. `createManifest` preserves both fields.

**`src/core/config.mjs` — Configurable compact threshold (CC 2.x parity)**
- `DEFAULT_COMPACT_THRESHOLD = 0.85` (raised from hardcoded 0.80)
- `getCompactThreshold()` — reads `compactThreshold` from user config with bounds check
- `setCompactThreshold(val)` — persists to config, validates 0.1–1.0 range

**`src/tui/claude/main.mjs` — Plan approval compaction (CC 2.1.x parity)**
`ConversationApp` registers an `onPlanApproved` callback (Section 8) that:
1. Immediately replaces messages with a `[Plan approved: <title>]` summary message
2. Runs `compactMessagesWithAi` on the previous messages to preserve key context
3. Silent on compaction failure — keeps the simple summary

**`src/tui/claude/main.mjs` — Skill hot-reload wired (CC 2.1.0 parity)**
`ConversationApp` effect (Section 9) calls `startSkillsHotReload(cwd)` on mount and
registers `onSkillsChanged(() => invalidateSkillsCache())`, so skills updated on disk
are picked up immediately. Cleans up watchers on unmount.
