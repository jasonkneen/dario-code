  <behavior>
    - SET-06: When `--setting-sources user,project` is passed, only user and project settings are loaded (managed, local, CLI layers skipped)
    - SET-06: When `--setting-sources` is not passed, all 5 sources load (default behavior)
    - SET-07: When `--settings '{"key":"val"}'` is passed (inline JSON starting with `{`), it is parsed and injected as CLI-level settings
    - SET-07: When `--settings /path/to/file.json` is passed (not starting with `{`), the file is read and parsed as CLI-level settings
    - SET-07: When `--settings` is not passed, no CLI-level settings are injected
  </behavior>
  <action>
    **In cli.mjs:**

    1. Add two new Commander options BEFORE the `.action()` call:
       ```javascript
       .option('--setting-sources <sources>', 'Comma-separated setting scopes to load (user,project,local,cli,managed)')
       .option('--settings <json>', 'Load settings from JSON file path or inline JSON string')
       ```

    2. In the `.action()` handler, BEFORE the dynamic `import('./src/main.mjs')` call, add:
       ```javascript
       if (options.settingSources) {
         const { setSettingSources } = await import('./src/core/config.mjs')
         setSettingSources(options.settingSources.split(',').map(s => s.trim()))
       }
       if (options.settings) {
         const { setCliSettings } = await import('./src/core/config.mjs')
         let parsed
         if (options.settings.startsWith('{')) {
           parsed = JSON.parse(options.settings)
         } else {
           const { readFileSync } = await import('fs')
           parsed = JSON.parse(readFileSync(options.settings, 'utf8'))
         }
         setCliSettings(parsed)
       }
       ```

    **In tests/settings-hierarchy.test.mjs:**

    Add a new describe block "CLI flag integration" with tests:

    - Test `setCliSettings({customKey: 'cliValue'})` then `loadSettings()` includes `customKey: 'cliValue'` at CLI precedence (overrides project but not managed).

    - Test that `setSettingSources` with an empty call resets to default (all sources).

    After tests pass, commit: `feat(01-02): wire --setting-sources and --settings CLI flags`
