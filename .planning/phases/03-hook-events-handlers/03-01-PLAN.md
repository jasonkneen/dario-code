<feature>
  <name>Multi-type hook handler dispatch</name>
  <files>src/core/hooks.mjs, tests/hook-handlers.test.mjs</files>
  <behavior>
    HTYP-01 (HTTP handler):
    - executeHttpHook(handler, context, verbose) POSTs JSON to handler.url (falls back to handler.command[0])
    - Request body includes hookType, toolName, input, output, sessionId
    - Uses AbortController for timeout (handler.timeout or DEFAULT_TIMEOUT)
    - Parses JSON response for action/message/reason/modifiedInput
    - On fetch error or timeout, returns { success: false, action: 'continue' }

    HTYP-02 (Prompt handler):
    - executePromptHook(handler, context, verbose) sends handler.prompt (falls back to handler.command[0]) to Claude
    - Uses handler.model or defaults to 'claude-haiku-4-5-20251001'
    - Sends context JSON with the prompt, asks for {"decision": "allow"|"deny"} response
    - Maps "deny" to action: 'block', "allow" to action: 'continue'
    - On API error, returns { success: false, action: 'continue' }

    HTYP-03 (Agent handler):
    - executeAgentHook(handler, context, verbose) spawns a read-only subagent
    - Uses AgentType.EXPLORE config (Read/Grep/Glob tools only)
    - System prompt from handler.command[0], input from context JSON
    - Returns agent output as message
    - On error, returns { success: false, action: 'continue' }

    HTYP-04 (Async mode):
    - When handler.async === true and handler.type === 'command', dispatchHook fires and forgets
    - Returns immediately with { success: true, action: 'continue' }
    - Errors logged to stderr in verbose mode only
    - Non-command handler types with async flag are NOT supported (ignored, runs sync)

    normalizeHandler updates:
    - Preserves async: handler.async ?? false
    - Preserves url: handler.url ?? null (for HTTP hooks)
    - Preserves prompt: handler.prompt ?? null (for prompt hooks)
    - Preserves model: handler.model ?? null (for prompt hooks)

    dispatchHook updates:
    - Check async mode first (fire-and-forget for command type)
    - Switch on handler.type: 'http' -> executeHttpHook, 'prompt' -> executePromptHook, 'agent' -> executeAgentHook, 'command'/default -> existing executeHook path
    - Attach statusMessage to result (existing behavior preserved)
  </behavior>
  <implementation>
    1. Update normalizeHandler() to preserve async, url, prompt, model fields
    2. Create executeHttpHook() using fetch with AbortController
    3. Create executePromptHook() using dynamic import of getClient
    4. Create executeAgentHook() using dynamic import of subagent
    5. Rewrite dispatchHook() as type-based switch router with async handling
    6. Export new functions for testability
  </implementation>
</feature>
